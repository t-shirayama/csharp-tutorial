## 到達目標

- 変更に強いコードの形を説明できる。
- テストしにくい設計を見分けられる。
- 実務のレビュー観点として設計原則を使える。
- DI lifetime、設定、ログ、例外処理を ASP.NET Core の実装に落とし込める。
- Controller、Application Service、Domain、Repository の責務分担を説明できる。
- DTO と Domain Model を分け、設計レビューで確認すべき観点を説明できる。
