# Transaction

## 目的

複数の DB 操作をまとめて成功または失敗にするトランザクションを理解します。

## 要点

- 関連する複数更新はトランザクションで守ります。
- 途中で失敗したら rollback します。
- EF Core の `SaveChanges` は通常1回の呼び出し内でトランザクションを使います。
- 注意: トランザクション中に長い外部 API 呼び出しを行うと、ロック時間が延びて障害の影響範囲が広がります。
- 注意: 失敗時の rollback、ログ、分離レベルの方針を決めずに実装すると、復旧やロック問題の調査が難しくなります。

## 実務での使い方

注文作成と在庫減算、支払い記録と請求更新など、片方だけ成功すると困る処理で使います。外部 API と DB の整合性は別途設計が必要です。
