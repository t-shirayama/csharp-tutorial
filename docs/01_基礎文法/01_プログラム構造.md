# C# のプログラム構造

## 目的

C# の最小プログラムがどのように実行されるかを理解し、`Program.cs` を読めるようにします。

## 前提

- .NET SDK が使える
- `dotnet new console` でプロジェクトを作成できる

## 要点

- C# のコンソールアプリは `Program.cs` から始まります。
- 近年の C# では top-level statements により、`Main` メソッドを書かなくても最小プログラムを作れます。
- 実務では古い書き方と新しい書き方の両方を読む可能性があります。
- 注意: top-level statements は、1つのプロジェクト内で複数ファイルに分けて書けません。
- 注意: 大文字と小文字は区別されます。`console.writeline` ではなく `Console.WriteLine` です。
- 注意: 文の末尾には基本的にセミコロン `;` が必要です。

## top-level statements の例

```csharp
// この例では「C# のプログラム構造」の要点を最小のコードで確認します。
Console.WriteLine("Hello, C#!");
```

この書き方では、ファイルの先頭に直接処理を書けます。コンソールアプリの入門や小さなツールでよく使われます。

## Main メソッドを明示する例

```csharp
// この例では「C# のプログラム構造」の要点を最小のコードで確認します。
internal class Program
{
    private static void Main(string[] args)
    {
        Console.WriteLine("Hello, C#!");
    }
}
```

既存の業務コードや古いプロジェクトでは、この形を見かけることがあります。

## 実務での使い方

実務では、アプリの入口がどこかを把握することが調査の第一歩です。コンソールアプリでは `Program.cs`、Web API では `Program.cs` と DI 設定、古いアプリでは `Main` メソッドを探します。
