# 遅延評価

## 目的

LINQ がいつ実行されるかを理解し、意図しない再実行を避けられるようにします。

## 要点

- 多くの LINQ は定義した時点では実行されません。
- `foreach`, `ToList`, `Count` などで列挙されたときに実行されます。
- DB クエリや重い処理では、実行タイミングが特に重要です。
- 注意: LINQ を定義した時点で実行済みだと思い込む。
- 注意: 同じ `IEnumerable<T>` を複数回列挙する。
- 注意: DB から全件取得した後にメモリ上で絞り込む。

## コード例

```csharp
// この例では「遅延評価」の要点を最小のコードで確認します。
var numbers = new List<int> { 1, 2, 3 };

var query = numbers.Where(number => number >= 2);

numbers.Add(4);

foreach (var number in query)
{
    Console.WriteLine(number); // 2, 3, 4
}
```

`query` を作った時点では評価されていないため、後から追加した `4` も含まれます。

## ToList で確定する

```csharp
// この例では「遅延評価」の要点を最小のコードで確認します。
var result = numbers.Where(number => number >= 2).ToList();
```

`ToList` を呼ぶと、その時点の結果が List として確定します。

## 実務での使い方

EF Core では LINQ が SQL に変換されるため、どこで `ToListAsync` するかが性能に影響します。メモリ上の LINQ でも、複数回列挙すると同じ処理が繰り返されることがあります。
