## 到達目標

- 要件を小さなタスクに分解できる。
- 動くものを作り、動作確認手順を残せる。
- 実装後にテスト、ログ、例外処理を追加できる。
- PR とリリースを想定した完成度確認ができる。
- API、DB、認証の実装を段階的な task に分け、確認コマンドと完了条件で進められる。
