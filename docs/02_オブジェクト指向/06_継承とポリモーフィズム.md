# 継承とポリモーフィズム

## 目的

継承とポリモーフィズムの基本を理解し、使いどころと避けどころを判断できるようにします。

## 要点

- 継承は、既存クラスを元に新しいクラスを作る仕組みです。
- ポリモーフィズムは、同じ呼び出しで異なる振る舞いを実行できる性質です。
- 実務では、継承より interface や合成の方が扱いやすい場面が多いです。
- 注意: 「似ている」だけで継承する。
- 注意: 親クラスに共通処理を詰め込みすぎる。
- 注意: 継承階層が深くなり、どこで何が実行されるか追いにくくなる。

## コード例

```csharp
// この例では「継承とポリモーフィズム」の要点を最小のコードで確認します。
var notifications = new Notification[]
{
    new EmailNotification(),
    new SmsNotification()
};

foreach (var notification in notifications)
{
    notification.Send("Hello");
}

public abstract class Notification
{
    public abstract void Send(string message);
}

public class EmailNotification : Notification
{
    public override void Send(string message) => Console.WriteLine($"Email: {message}");
}

public class SmsNotification : Notification
{
    public override void Send(string message) => Console.WriteLine($"SMS: {message}");
}
```

## 実務での使い方

フレームワークの拡張点や、明確な親子関係があるモデルで使います。一方、業務ルールの変更に追従するには、継承階層が深い設計は硬くなりがちです。
